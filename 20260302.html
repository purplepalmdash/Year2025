风格复原强制优先级（必须严格按此顺序权衡，任何冲突都优先前项）：

0. 行数一致性（最高优先级约束）：
   - 重建版必须逐行对应原版行数，同一列的行数误差 ≤1 行
   - 采用“逐行扫描 + 逐行重建”模式：禁止使用自动换行、flowable 或段落自动布局
   - 先通过 OCR 的 y 坐标分布（或 bbox top/bottom 聚类）统计原页每列的视觉行数
   - 重建时每列强制绘制固定行数：缺失行补空行（或适当空白字符），超出行尝试合并短行或微调行距
   - reportlab 绘制使用固定 y 坐标递减（y -= fixed_line_height），fixed_line_height 根据原页平均行距反推

1. 界栏/矩形方框检测与绘制：
   - 如果 PaddleOCR-VL 输出中包含 border / frame / rectangle / ruling / line 类型，或多个文字块的 bbox 形成封闭包围，则必须绘制细黑矩形框
   - 绘制方式：reportlab canvas.rect(x1, y1, x2-x1, y2-y1, fill=0, stroke=1, width=0.6–1.0)
   - 坐标取所有被包围文字块的 min_x/min_y/max_x/max_y，向外扩展 3–6pt 作为边距
   - 如果 OCR 未明确标记，但视觉上明显有界栏（外接矩形明显包围多行文字），也视为需要绘制框
   - 鱼尾、卷首、界线同等处理为矩形或直线

2. 竖排方向与阅读顺序：
   - 整页从右往左排列列（古籍右起左行）
   - 每列从上到下
   - 严格遵循 PaddleOCR-VL 的 reading_order，从最右侧列开始逐列向左
   - 竖排文字使用 canvas.rotate(90) 或自定义坐标变换实现

3. 注释/夹注/双行小字/眉批位置：
   - 双行小字 / 注文 尽量放在正文右侧或行间夹注位置，字号缩小到正文的 60%–75%
   - 眉批放在页面顶部或对应正文上方，字号缩小到 50%–65%
   - 根据 block type 或 bbox 位置判断（右侧/上方/行间）

4. 字号层级：
   - 正文基准 14–18pt
   - 注文/夹注 10–13pt
   - 眉批/小字 8–11pt
   - 根据 bbox 高度或 type 粗估

5. 行距与字距：
   - 行距紧凑（1.05–1.15 倍行高）
   - 字距均匀（0.92–1.0 倍），避免现代宽松间距

6. 分段/章节/圈点：
   - 保留原版明显的段落空行、界栏位置、鱼尾/圈点位置近似
   - 圈点可用小圆点或符号模拟

7. 字体选择：
   - 正文优先 SimSun / STSong（宋体）
   - 注文/注释优先 KaiTi / SimHei（楷体或黑体）
   - 确保繁体/古字/异体字不出现方块

8. 背景：
   - 纯白色（或极浅米色 #FAF9F2），无任何纹理/图像
