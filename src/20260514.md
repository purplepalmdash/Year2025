### 3. lv extend/shrink
Via:      

```
# 查看当前 LVM 情况
lvdisplay
vgs
lvs
df -h

# 从 /home 缩小 200G 给根分区（示例）
# 先缩小 /home 的逻辑卷和文件系统
lvreduce -L -200G -r /dev/centos/home     # -r 会自动 resize 文件系统（CentOS 8+ 推荐）

# 再扩大 root
lvextend -L +200G -r /dev/centos/root
```

### 5. mtp model startup
Startup command:     

```
CUDA_VISIBLE_DEVICES=0 ./build/bin/llama-server   -m /home/dash/models/Qwen3.6-35B-MTP-GGUF/Qwen3.6-35B-A3B-UD-Q4_K_XL.gguf   --mmproj /home/dash/models/Qwen3.6-35B-MTP-GGUF/mmproj-F16.gguf   --host 0.0.0.0   --port 8080   --ctx-size 262144   -ngl 99 
  --flash-attn on \
  --cache-type-k q8_0 \
  --cache-type-v q8_0 \
  --jinja \
  -t 0 \
  --no-mmap \
  --spec-type draft-mtp \
  --spec-draft-n-max 3 \
  -np 1 --cont-batching off --no-warmup
```
### 6. MTP llama.cpp
via:     

```
cd Code
git clone -b mtp-clean https://github.com/am17an/llama.cpp.git
# 或者官方 PR：
# git clone https://github.com/ggml-org/llama.cpp.git

# git fetch origin pull/22673/head:mtp
# git checkout mtp

cd llama.cpp
cmake -B build -DBUILD_SHARED_LIBS=OFF -DGGML_CUDA=ON   # CPU/Metal 就把 CUDA=OFF
cmake --build build --config Release -j --clean-first --target llama-cli llama-server
```
